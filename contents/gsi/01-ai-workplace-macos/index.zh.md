# 如何配置一个属于你的 AI 工作台（macOS 版）

## Chatbot vs Agent

在浏览器里问 AI 很快，但真要改你电脑上的项目、批量处理文件，并让助手始终对准当前文件夹里的内容，你需要一种以本地目录为中心的工作方式：文件在磁盘上，由编辑器里的助手按路径读写，而不是靠一次次复制粘贴。

---

### 网页 AI vs 本地 AI 工作台：到底差在哪？

| 维度          | 网页对话型 AI        | 本地工作台（VS Code + Agent） |
| ----------- | --------------- | ---------------------- |
| **读你的项目文件** | 通常要手动复制粘贴或一次次上传 | 直接打开文件夹，Agent 按路径读写    |
| **批量改文件**   | 很难稳定、可复查地改一整套目录 | 适合多文件重构、重命名、生成脚手架      |
| **「项目记忆」**  | 会话一关，上下文容易断     | 工作区即边界，指令与文件结构天然对齐     |
| **可重复流程**   | 依赖你每次描述环境       | 终端、Git、扩展可脚本化、可团队对齐    |
| **上手成本**    | 低               | 稍高，但一次搭好长期省力           |

VS Code 是这类工作台里很常见的一种入口；装好相应的编程助手扩展后，你可以在同一个窗口里改文件并让助手协同，把日常写作与协作收拢到一处。

---

### VS Code 是什么？为什么推荐 VS Code？

Visual Studio Code 本质上是一个用来查看、编辑和管理你电脑中文件的工作界面，而不是文件真正存放的地方。可以把电脑想成一个房间，所有文件都安静地放在文件柜里；VS Code 更像是一张桌面，当你打开它时，只是把需要的文件拿出来摊在桌面上进行操作。你在桌面上做的任何修改，比如编辑、保存或删除，都会直接影响文件柜里的原始文件。

从这个角度看，VS Code 把文件浏览、搜索和文本编辑整合在同一套界面里，让你不必在多个应用之间来回切换。它还可以通过安装扩展不断补强能力；例如常用的 Claude 编程助手扩展可以直接装在 VS Code 里，把自己写代码与让助手协助改文件收拢在同一环境里。

## 第二部分：在你的电脑上管理工作区

### 基本文件管理规范

很多初学者在刚用 VS Code 时，最大的困扰往往不是不会点菜单，而是文件越堆越乱、路径记不清、甚至把不同项目的材料混在一起。下面几条习惯能帮你避开大部分麻烦。

先在你的电脑上划一块专门放工作的区域，例如建一个名为 Workspace 的总目录，下面用两个子目录分清「全局规则」和「具体项目」。`system` 用来放希望所有项目都遵守的约定，例如文章体例、任务流程说明等，相当于给 AI 助手一份长期有效的「工作说明」。`project` 则用来放一个个独立任务，每个任务各自占一个子文件夹，里面可以再按需要加说明或配置。目录可以长成下面这样，具体名字可按你的习惯微调，关键是层次要一眼能看懂。

```
Workspace/  
├── system/  
├── project/  
│ ├── subproject1/  
│ ├── subproject2/  
│ └── subproject3/
```

在此基础上，坚持「一个项目一个文件夹」：不同任务的文件不要混在同一层目录里，否则后期整理和搜索成本会陡增。命名时尽量用能说明用途的名字，例如 `report.md`，少用 `test1`、`aaa` 这类临时称呼。

**关于同步与备份**：助手读取的是你本机磁盘上、当前工作区里的文件；若资料只存在于网页网盘且未同步到本地，它无法像打开本地文件夹那样直接访问。若你用 Git 管理 Workspace，请避免把含密钥、令牌或私密配置的目录推送到公开仓库；使用网盘同步时也要注意冲突和权限，以免误覆盖或泄露。

养成上述习惯后，查找与维护会轻松很多。结构不必第一天就做到完美，但有章可循一定比完全随意堆放更省力。

> [!TIP]
> 
> 记住：VS Code 只是帮你操作文件的工具，真正的文件始终在你的磁盘上。清晰的目录结构会在以后省下大量时间。

### 打开你的 Workspace

依次点击顶部菜单 **File → Open Folder**，选择你的 Workspace 或项目根目录。

![](images/01-vscode-usage-20260305172317.jpg)

打开后，在资源管理器中可展开或收起文件夹，并用工具栏新建文件、新建文件夹、刷新视图。

![](images/Pasted%20image%2020260330171358.png)

### 创建文件夹和文件

#### 创建新文件夹

在侧栏中选中 `project`（或你用于放项目的目录），点击工具栏上的新建文件夹按钮，为第一个项目建子目录。

![](images/Pasted%20image%2020260330171538.png)
#### 创建新文件

同样可在目标文件夹下用工具栏新建文件。

![](images/Pasted%20image%2020260330171713.png)

新文件默认为空，保存时请写上后缀名，例如 `test.md`。

![](images/Pasted%20image%2020260330171735.png)

创建完成后，在侧栏点击文件即可查看或编辑。

![](images/Pasted%20image%2020260330171756.png)

### 重命名和删除

#### 重命名文件

右键文件并选择 **Rename**，输入新名称即可。

> [!TIP]
>
> 在 Mac 上也可先选中文件，再按 Enter 快速重命名。

![](images/Pasted%20image%2020260402112911.png)
#### 重命名文件夹

操作方式与文件相同：右键选择 **Rename**，或选中后按 Enter。

![](images/Pasted%20image%2020260330171934.png)

#### 删除文件或文件夹

右键目标项，选择 **Delete** 即可删除。

![](images/Pasted%20image%2020260330171954.png)

### 搜索面板

搜索会在当前工作区（含子文件夹）内查找匹配文本，结果会高亮；点击左侧文件名可跳到对应位置。

![](images/Pasted%20image%2020260330172051.png)

### 扩展市场

扩展视图即插件市场，可在这里为 VS Code 增加功能。在搜索框输入 `claude`，找到 **Claude Code for VS Code** 并安装。

![](images/01-vscode-usage-20260317182540.jpg)

安装完成后，左侧活动栏会出现 Claude 相关图标。点击后会出现 Claude Code 的管理区域。

![](images/Pasted%20image%2020260330161852.png)

若要新开一个会话，可在 Claude Code 面板里点击 **New Session**，也可点击编辑器右上角的 Claude Code 图标，两种方式任选其一即可。

![](images/Pasted%20image%2020260330161706.png)

![](images/Pasted%20image%2020260330162017.png)

## 第三部分：重要文件与目录说明

在管理文件时，还有一类平时不太显眼、却经常影响环境是否稳定的内容：隐藏目录与环境配置文件。弄清它们各自做什么，能帮你少踩坑、排错也更快。依次点击菜单 **Go → Home** 进入用户主目录。

![](images/Pasted%20image%2020260330172747.png)

默认情况下，访达里通常只看到 Desktop、Documents、Downloads 等常见文件夹。

![](images/Pasted%20image%2020260401152144.png)

按下 `Command + Shift + .` 可显示隐藏项。与编程助手常见相关的包括：用户目录下的隐藏文件夹里，往往有登录状态、偏好设置，以及会在多个项目之间共享的补充说明、外部工具连接方式等；另有一些清单型文件用来登记本机已启用的能力。Shell 启动脚本或项目里的环境变量文件则常用来存放别名或密钥引用。这些位置往往涉及敏感信息，请勿截图外传或提交到公开仓库。

![](images/Pasted%20image%2020260401152206.png)
